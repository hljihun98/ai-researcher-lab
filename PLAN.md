## 1. 현재 상태 한눈에

- **백엔드**: Flask 웹앱, Gemini(`gemini-2.0-flash-lite` 기본) + 라이트 모드(세션 5호출),
  키 콤마 로테이션(429 시 다음 키), grounding(팩트체커 실검색), 오류 상태 분리,
  세션 저장/목록/조회/내보내기/공유(브라우저 소유권 쿠키), 평가 하네스(`eval/`).
- **프론트**: 캔버스 픽셀 오피스(BFS 이동·상태머신), 반응형, 상시활동(배회/일하는 척),
  마주침 이모지 인사, 캐릭터 개성(색조), 라이트/다크, 사운드, 지난연구 재생.
- **가장 큰 미완**: "질문 → 백엔드가 다 계산한 뒤 → 애니메이션 재생" 구조라,
  **기다리는 동안(스피너)은 과정이 안 보인다.** 사용자가 원한 "기다림=관람"이 아직 아님.

## 2. 다음 추천 (지금 진행) — ⭐ 실시간 스트리밍 "기다림 = 관람"

**목표**: 백엔드가 발언을 만들어내는 즉시 프론트로 흘려보내, 연구원들이 **실시간으로**
자리에서 일어나 이동·대화하는 걸 보게 한다. 무료 등급은 호출이 느려(수 초+429 대기)
오히려 스트리밍이 그 시간을 "관람 시간"으로 바꿔줘 체감이 크게 좋아진다.

**설계 (이벤트 스트림, SSE)**
- 백엔드: `POST /api/run/stream` → `text/event-stream`. 세션 루프를 **제너레이터**로
  바꿔 발생 즉시 이벤트를 방출:
  - `round`  : {index, agents:[a1,a2], location, confidence}
  - `utterance` : {agent, message, confidence, location, turn, responds_to}  (agent.speak 직후)
  - `final`  : {final_answer, confidence_score, status, has_errors, id}
  - `error`  : {message}
- 기존 `POST /api/run`(일괄)과 세션 저장/`/api/session/*`(재생·공유)은 **그대로 유지**.
- 프론트: `run()`이 스트림을 소비 → `round`에 `PixelOffice.encounter()`, `utterance`에
  `showSpeech()+타이핑`, `final`에 `finalize()`+답변. 지난연구 재생은 기존 일괄 경로 재사용.

**작업 분담**
- **ChatGPT (백엔드)**: `/api/run/stream` + 제너레이터 리팩터(라이트/풀·deadline·오류·
  세션 저장·id 유지). 계약/테스트 → `CHATGPT_HANDOFF_STREAM.md`에 준비 완료.
- **Claude (프론트)**: SSE 소비로 `run()` 개편(엔진 실시간 구동), 스트림 미지원/실패 시
  기존 일괄 경로로 폴백. 백엔드 계약이 배포되면 바로 얹음.

**리스크/주의**: gunicorn 버퍼링(헤더 `X-Accel-Buffering: no`, flush 필요), Render SSE 지원,
워커 1개(인메모리 세션 일관성)와 궁합 OK. 실패 시 일괄 경로 폴백으로 항상 안전.

## 4. 권장 진행 순서

1. **실시간 스트리밍** (P0) — GPT가 SSE 백엔드 → Claude가 프론트 실시간 소비.
2. **대화 다양화/구조** (P0, 핸드오프 됨) — GPT.  ※1과 병행 가능(백엔드 서로 다른 영역).
3. **평가 하네스 실측** (P1) — 쿼터 여유(유료/리셋) 생기면 `python -m eval.harness --n 3`.
4. **Phase 4 사용자 개입** (P1) — 스트리밍이 있으면 "진행 중 개입"이 자연스러워짐.
5. 이후 P2/P3에서 선택.

## 5. 상시 주의(계약)
- `/api/run` 응답 스키마·`responds_to==null`(라운드 시작) 규칙·엔진 공개 API 유지.
- 실제 키/비밀은 코드/커밋 금지(환경변수만). 데모 모드·CI(우분투)로 항상 검증.
- 로컬에 Python 미설치 → 백엔드는 CI, 프론트는 Node 문법검사 + 배포 확인으로 검증.
